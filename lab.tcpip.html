OSI Model

Physical

This layer includes physical mediums, such as cards and cords, used to send and receive data in the form of electrical bits. Protocols include Fast Ethernet, RS232 and ATM.

Data Link

In this level individual packets are encoded/decoded into bits and handles error correction in the physical layer as well as flow control and frame syncronization. Protocols include PPP, FDDI, ATM and IEEE.

Network

The Network layer provides switching and routing for transmitting data from node to node with the use of virtual paths. Protocols include AppleTalk DDP, IP and IPX.

Transport

The fourth layer handles the transfer of data between hosts and deals with end to end error recovery and flow control. Protocols include SPC, TCP and UDP.

Session

The Session layer establishes, manages and terminates connections between host applications. Protocols include NFS, NetBios, RPC and SQL.

Presentation

The sixth layer translates data between the application format and the network format. It also encrypts data before sending it across the network. Protocols include ASCII, TIFF, GIF, JPEG and MPEG.

Application

The last layer supports application and end user processes. Here everything is application specific such as file transfers and emails. Some applications, Telnet and FTP, exist entirely in the application layer. Protocols include WWW, NFS, SNMP, Telnet, HTTP and FTP.

TCP/IP Model

Network Access Layer

The first layer of the TCP/IP model is the Network Access Layer. It is the lowest layer on the model providing an interface for the physical network. The physical network is made up of a physical medium such as an Ethernet, coaxial or fiber optic cable. This layer defines how data will be sent physically through the physical medium in the form of electrical pulses. It will also receive these electrical pulses, translate them and handle error correction in the event of corrupted data. The most popular architecture for this layer is Ethernet which uses CSMA/CD discussed in more detail below.

CSMA/CD

Carrier Sense Multiple Access/Collision Detection

Internet Layer

The second layer in the TCP/IP model is the Internet Layer. This layer wraps data packets from the higher levels into frames, otherwise known as IP datagrams. These frames contain the source and destination IP addresses which allow routers to correctly transmit the frames to the appropriate destination. This layer also allows host systems to insert datagrams into the network independent of other datagrams for easier delivery to the destination hosts. This injection ability keeps the network medium from being monopolized by a single system. Since the datagrams are independent they may arrive at the destination out of order. It is the responsibility of the higher layers to reassemble the datagrams in the proper order to reconstruct the file. This layers' main protocols include the following list:
  • IP (Internet Protocol)
  • ICMP (Internet Control Message Protocol)
  • ARP (Address Resolution Protocol)
  • RARP (Reverse Address Resolution Protocol)
  • IGMP (Internet Group Management Protocol)
 

Transport Layer

The third layer in the TCP/IP model is the Transport Layer. Here the system devices gain access to communicate with other peers and servers on the network.The main protocols include:
  • TCP (Transport Control Protocol)
  • UDP (User Datagram Protocol)

Application Layer

The fourth and final layer in the TCP/IP model is the Application Layer. Here programs like internet browsers or video games will reside and communicate with the lower layers to send and receive network traffic. This layer defines how these programs will interact with the transport layer. The main protocols include:
  • DNS (Domain Name Service)
  • HTTP (Hypertext Transfer Protocol)
  • Telnet
  • SSH (Secure Shell)
  • FTP (File Transfer Protocol)
  • SNMP (Simple Network Management Protocol)
  • SMTP (Simple Mail Transfer Protocol)
  • DHCP (Dynamic Host Configuration Protocol)
  • X Windows
  • RDP (Remote Desktop Protocol)
